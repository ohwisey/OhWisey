HOW IT WORKS
Build your own dashboard in 5 minutes ›

You don't need ten apps. You need one folder. This dashboard is the hub — each standalone is a single HTML file in its own folder right next to it.

  1. Fork the repo. Drop index.html into any folder, or fork the GitHub repo to start from this exact template.
  2. Build a standalone. One HTML file, inline CSS, inline JS, localStorage (or Supabase) for state. No bundler. No npm. Host it anywhere — sibling folder, GitHub Pages, Vercel, your own domain.
  3. Add it to the dashboard. Open index.html, find the TILES array at the top of the script, add a new entry with name, icon, url, desc.
  4. Push to git. git add . && git commit -m "add my standalone" && git push. Vercel redeploys in ~20s.
  5. Open it. Click the tile. Your standalone runs on its own — data either local or synced via Supabase.

WHY THIS WAY

Most dashboards let you "add tiles" through a UI that writes to a database. That database becomes the lock-in. Here, the tile list lives in code — version controlled, forkable, readable. Adding a tile is the act of pushing to git. Newbies learn the workflow by doing it once.

OPTIONAL: SUPABASE FOR YOUR STANDALONES

The dashboard signs you into Supabase. Your standalones inherit that session via window.opener?.ohwisey?.session — meaning workout logs, sleep data, etc. sync across devices without each standalone needing its own auth flow.
