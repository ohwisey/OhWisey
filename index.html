HOW IT WORKS
Build your own dashboard in 5 minutes ›

You don't need ten apps. You need one folder. This dashboard is the hub — each standalone is a single HTML file in its own folder right next to it.

  1. Save this file. Drop index.html into any folder on your computer. That's your hub.
  2. Build a standalone. Make a folder like workout-logger-standalone/ next to index.html and put your app's index.html inside. One file. Inline CSS, inline JS, localStorage for state. No build step.
  3. Link it. Tap the ⚙ gear → Add standalone. Give it a name, an emoji, and the path (e.g. ./workout-logger-standalone/index.html).
  4. Open it. Click the tile. The standalone runs on its own — your data stays in the browser.
  5. Ship it. Push the folder to GitHub, deploy on Vercel free tier, share the link.

SYNC ACROSS DEVICES

By default the dashboard works fully offline — tiles save to your browser's localStorage. To sync across phone + laptop + iPad: hit the gear icon above, paste your Supabase URL and anon key, and sign in with magic-link email.

Don't have Supabase yet? See SETUP.md — 10-minute walkthrough that creates the project, runs the schema, and gets you signed in.

ONE TIP

Namespace your localStorage keys per standalone (e.g. workout.log.2026-05-15) so two apps in the same browser never collide.
